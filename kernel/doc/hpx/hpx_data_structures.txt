CCB: Connection control block,in each endpoint and created in phase of initial access,used to save connection information with gateway; 主要字段：status，HPX address/domain，gateway HPX address，gateway MAC address，encryption key，refresh timer，...
ECB: Endpoint control block, in gateway and used to save connection information for each endpoint associated with it. 包含的参数：endpoint的MAC地址，ECB的状态（KEY_NEGOTIATING，...），HPX address/HPX domain，encryption key，refresh timer（一旦这个时间超时，那么ECB会被删除），......
SCB: Session control block, corresponding each session between 2 endpoints。包含的参数：类型（CONNECTION-ORIENTED or CONNECTIONLESS），owner process/thread handle，client HPX address/HPX domain，server HPX address/HPX domain，session ID，client seq no/ack，server seq no/ack received，丢包计数（CONNECTIONLESS），status(REQUEST_RECV,COMPLETED,CLOSE_HALF），接收数据缓冲区指针，接收数据缓冲区长度，当前接收数据缓冲区占用长度，发送数据缓冲区，发送buff长度，当前发送buff长度，pending的线程列表，spin_lock，sending list（CONNECTION-ORIENTED类型），...
Routing table: Same likes IP's routing table, through wich the out going interface, next hop's hardware address,and other information could be retrieved by giving the domain address;
AWL：Accept White List, in each endpoint running HPX protocol, contains the hardware address of endpoints or gateways where the endpoint can accept frame from.The frame beyond the AWL will be droped directly;
DBL：在gateway上的黑名单（Deny Black List），记录了不合法的endpoint的MAC地址。一旦收到广播报文，首先查询DBL。如果源MAC地址在DBL里面，则直接丢弃该广播。这是为了阻止恶意endpoint发起连续的密钥协商，或者发起连续的广播攻击而设置的；
SDB: Service Description Block, describes the current service(s) that an endpoint can offer. HPX protocol will match the SDB when incoming service request arrive and create a SCB for the request if it pass the matching。包含的参数：连接类型，process/thread ID，status（INITIALIZED/COMPLETED），service ID，HPX address(server side)，backlog，pending_req（当前尚未被服务的请求数，要小于backlog），SCB list，spin_lock，阻塞队列（调用accept的process/thread会阻塞在该SDB上，如果没有pending request等待服务的话），
